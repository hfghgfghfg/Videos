What Is Useful 

N HAIR SETUP Tool is a technical utility for creating dynamic curve setups in Maya, designed to automate and standardise the creation of dynamic curve simulations in a production environment. It eliminates tedious and error-prone manual node construction, significantly reducing common setup issues in complex workflows. At the same time, it enforces a unified and highly predictable node architecture and connection rules, ensuring stable collaboration between all components while preventing conflicts or inconsistencies caused by manual operations.

During the creation stage, the tool generates a fully deterministic, pipeline-compliant dynamic curve setup, consistently applying strict naming conventions and structural standards. This not only greatly improves the efficiency of artists but also enhances overall simulation stability and consistency. The resulting setups exhibit reliable, repeatable, and scalable behaviour, allowing them to be easily applied across different shots, assets, and large-scale productions, ultimately improving the productivity and output quality of CFX artists.

The N HAIR SETUP tool includes all the operational tools required to create a dynamic curve setup. These tools include:

  • Convert Yeti Groom(s) To Curve(s)

Start N Hair Setup



It is designed to automate the initial setup, generating a well-organised hierarchy of groups, controls, and nodes, ensuring consistency across projects and reducing the risk of human error.

A key feature is its ability to check for naming conflicts before setup. If any conflicts are found, the script notifies the user, preventing accidental overwrites or misconnected nodes.

The script creates essential nHair components, including the nucleus node, volume axis field, and key controls such as HAIR_CTRL and WIND_DIRECTION_CTRL. These controls come with predefined attributes for amplitude, speed, turbulence, and directional forces, all automatically connected to the corresponding nodes. This setup allows artists to focus on creative adjustments rather than tedious technical tasks.

Additionally, it organises the scene into functional groups and displays layers, simplifying scene management and improving visibility. Input and output curves, colliders, constraints, hair surfaces, and other elements are logically grouped and colour-coded for easy identification.

Overall, Start N Hair Setup provides a standardised, reliable, and pipeline-compliant foundation for dynamic hair setups, enhancing efficiency, collaboration, and simulation stability in production workflows.

Note: A warning dialogue window will appear if there are naming conflicts in the scene.


Usage

Click Start N Hair Setup to begin the first stage of creating a hairSystem dynamic curve setup.

Convert Yeti Groom(s) To Curve(s)



This converts Yeti groom strands into NURBS curves, allowing you to create guide curves from an existing groom.

It simplifies the workflow by eliminating the need to manually recreate guide curves while maintaining accurate strand placement for hair simulations. The generated curves can be further refined or used directly in dynamic hair setups. If needed, you can later use the Reset Curves tool to rebuild the curves.


Note: This is a shortcut to the same function in the Yeti menu.


Usage

Select the groom in Maya scene before clicking the button.

Generate Dynamic Curve(s) On Curve(s)





The tool will create hairSystem dynamic curves, generate input and output curves, apply standardised naming and grouping, and establish all necessary connections. Each output curve has an input_curve_weight blendShape attribute. You can modify the blendShape weights accordingly as needed.


Tip: When defining a label, if you name it "X", the generated hairSystem will be named "X_HS". When the hairSystem is created, a HairSystemShape_Set will also be generated, which includes all the hairSystem shapes.


Usage

After running Start N Hair Setup, select one surface object (if curve attachment is required) along with one or more NURBS curves in the scene. Then, assign a hairSystem label using the UI option menu. You may optionally enable the check-box options to customize the hairSystem behaviour.


Assign HairSystem

The function links selected NURBS curves to an existing hairSystem in the scene. You can either specify an existing label in the scene or create a new custom label. It allows multiple curves to be connected to a specific hairSystem, automatically creating follicles and connecting them to the hairSystem’s input and output attributes

It applies naming conventions that comply with the workflow standards and organises all follicles and curves under the corresponding hair system hierarchy. By automating these processes, it reduces the risk of human errors and maintains consistent scene organisation.


Attach Curve(s) to the selected mesh surface

The function binds the root of selected NURBS curves to the surface of a chosen mesh, establishing a direct connection that allows the curves to follow the mesh’s deformations and transformations for accurate placement in hair simulations. The attachment process automatically generates the necessary follicle nodes with correct UV mappings, ensuring the curves respond properly to dynamic mesh deformations.


Snap curve root(s) to the selected mesh surface

The function instantly positions the roots of selected NURBS curves directly on the surface of a target mesh. Unlike attachment, this operation is purely positional and does not create dynamic connections, making it ideal for aligning or correcting curve placement before binding to a hairSystem. The tool maintains proper curve orientation and ensures positional consistency.

Merge Hair Systems





The function allows users to combine multiple hairSystems in a Maya scene into a single, unified hairSystem. You can either specify the name of an existing hairSystem in the scene or create a new custom name. It also reorganises groups and reconnects associated nodes, making adjustments during the setup process more flexible and improving workflow efficiency.


Usage

Select hairSystems you want to merge by highlighting their buttons in the interface, ensuring that at least two hairSystems are selected, and specify a label for the resulting merged hairSystem, avoiding invalid characters or reserved terms (e.g., HSM).


Specify a label for the merging hairSystem

This feature lets users assign a label to the merged hairSystem, ensuring clear identification and consistent naming within the Maya scene and N HAIR SETUP hierarchy. Labels can only include letters, numbers, or underscores, and reserved terms like “HSM” are prohibited. The label is applied to the merged hairSystem transform, input/output curve groups, and follicles, with automatic validation to prevent conflicts or invalid entries. This helps maintain pipeline-compliant naming, avoids node errors, and keeps the scene organised for easier workflow.




Change Hair System On Curve(s)





The tool allows users to reassign selected output curves to a different hairSystem within a Maya scene. It validates curve selections, creates a new hairSystem when necessary, and updates all relevant node connections, enabling flexible redistribution of curves across hairSystems while preserving simulation integrity.


Usage

Select one or more output curves from the output_curve_GRP group. In the UI, choose an existing hairSystem or specify a new label to create one automatically. A warning dialogue will be displayed if the selected curves do not belong to the output_curve_GRP group.


Assign HairSystem To Curve(s)

The feature validates that only valid output curves are selected, then reconnects the associated follicles, input/output curves, and blendShape nodes to the target hairSystem. If the specified hairSystem does not already exist, it is automatically created and correctly connected to the nucleus and time nodes. All affected nodes are renamed and reorganised to follow pipeline-compliant naming and hierarchy rules, ensuring stable simulation behaviour, preventing connection conflicts, and enabling flexible redistribution of curves across hairSystems in complex grooming setups.




Replace Input Curve





The tool allows users to non-destructively replace an existing input curve in an N Hair Setup with a new curve, while preserving all downstream connections, including follicles, rebuildCurve nodes, blendShapes, and output curves. This enables artists to refine or redesign guide shapes without rebuilding the hairSystem or breaking simulation integrity.


Usage

First, select the current input curve from the input_curve_GRP and assign it using the Current Input Curve button. Then select a new curve in the scene and assign it using the New Input Curve button. Once both curves are specified, execute the replace operation to update the input curve while keeping the existing hair setup intact.


Current input curve

This field specifies the existing input curve connected to a follicle in the N Hair Setup and used to drive the hairSystem. Only one valid NURBS curve under input_curve_GRP can be assigned. The tool validates the selection to ensure it belongs to the correct follicle hierarchy, preventing invalid selections or unintended changes.


New input curve

This field specifies the curve used to replace the shape of the current input curve. The tool validates the selection as a single NURBS curve, duplicates it, and transfers its shape to the existing input curve while preserving all downstream connections. This allows guide curves to be reshaped without rebuilding the hairSystem.




Remove Curve(s) From Hair System(s)





The tool removes selected output curve(s) from an existing hairSystem in the N Hair Setup, along with their associated follicles and internal connections. It provides an option to duplicate the corresponding input curves before removal, allowing the user to preserve guide shapes for reuse or further editing.


Usage

Select one or more output curve transforms located under output_curve_GRP in the Maya scene.


Make duplicated curves

If “Yes” is checked, the feature preserves the input curve before deleting associated output curves in the N Hair Setup. It duplicates the input curves by copying their degree and CV positions, retaining the original guide shapes for reference or backup during hair adjustments.

Only valid NURBS curves under output_curve_GRP are duplicated. Duplicated curves are renamed with a duplicated_ prefix, preserving hierarchy and naming conventions to ensure pipeline compliance and prevent conflicts.




Replace Nucleus In N Hair Setup





The tool allows users to replace an existing Nucleus node with another one in a Maya scene using the N HAIR SETUP workflow. This operation automatically reconnects all relevant nodes—including hairSystem, nRigid, and dynamicConstraint nodes—as well as any connected HAIR_CTRL controllers to the new Nucleus node, ensuring simulation continuity and pipeline consistency.


Tips: This feature is especially useful when multiple N HAIR SETUP files exist within the same scene. Since each setup has its own independent nucleus, interactive dynamic simulations cannot occur. By sharing a single nucleus among multiple setups, the curves can interact dynamically.


Usage

Use the UI to select the existing Nucleus node under the H_HAIR_SETUP group to be replaced, then select the new Nucleus node that will take its place.


Replaced Nucleus

The currently active nucleus node in the N_HAIR_SETUP that needs to be replaced. Only one Nucleus node can be selected. The feature validates that it belongs to the N_HAIR_SETUP group.


Replacing Nucleus

The new Nucleus node will take over the connections. It cannot be the same as the replaced nucleus. The feature validates selection and prevents conflicts.




Apply Relative Nucleus (Editable)





The tool automates the creation and setup of a “Relative Nucleus” within an N Hair Setup, allowing users to control the dynamic behaviour of the nucleus. It sets up constraints, loft surfaces, curves from mesh edges, point-on-surface nodes, and associated transforms to drive the nucleus dynamically.

Especially during fast animation movements, dynamic simulations often result in exaggerated deformations. Increasing the relative value (Relative Nucleus Point / Relative Nucleus Point Orient) helps stabilise these deformations in the simulation.


Tips: You can update the Relative Nucleus at any time, which is very useful when simulating hairSystem dynamics.


Usage

Select a control or object in the scene with all translation and rotation attributes unlocked, then click ‘relative static controller’ to define the static controller.

Select two edges from a mesh, then click the ‘relative motion Controller’ button to define the motion controllers.

Click the Apply button to create or update the relative nucleus.


Relative static controller

The feature allows you to select a transform in the scene that serves as the static reference for the “Relative Nucleus”, providing a stable anchor for the nucleus and ensuring predictable dynamic behaviour when hair curves respond to motion.


Relative motion controller

The feature consists of two selected mesh edges that drive the dynamic motion of the “Relative Nucleus”. These edges provide positional and directional information, allowing the nucleus to respond naturally to mesh deformations or animated movements.

Override Colour On Curve(s)





By assigning distinct colours, the tool allows users to quickly identify different curve roles, such as input curves, output curves, guide curves, or debug references. This significantly improves scene readability.


Usage

Select curve(s) transform, and enable or disable colour on the curve(s) transform or the curve’s shape.


Specify color

The feature allows users to assign custom RGB colors to selected curves in the scene, enabling visual distinction between different types of curves. Users can double-click to set any color preference, and use 'Edit → Save Setting’ or ‘Edit → Reset Setting’ to save or remove the UI color settings.


Enable overrides

When enabled, the tool applies RGB color overrides to either the curve transform or the curve’s shape, based on the user’s selection.

When disabled, the override color will be removed from both the curve transform and shape.

Create IK Joints From Output Curve(s)





The tool generates IK joints along selected curves in the N Hair Setup workflow. It automates the creation of joints, IK spline handles, and locators for advanced twist control, providing a clean and organised rigging hierarchy.


Usage

Select output curve(s) or transforms/shapes in the Maya scene. Set the desired number of joints on the shortest and longest curves using the UI fields, and choose whether to connect the twist locators to follicles or a custom controller.


Constrain the World-Up Locator(s)

The feature enables users to control the orientation of IK spline handles along the curve by connecting the world-up locators to a reference object. It ensures consistent twist and rotation along the curve, preventing unwanted flipping or rotation artefacts.

Constraint Options:

To Follicles – Uses the corresponding follicle as the World Up Object to drive the orientation of the IK spline handle, ensuring consistent twist along the curve.

To Controller – Any user-defined transform can be used as the World Up Object to drive the orientation of the IK spline handle, ensuring consistent twist along the curve.

Create And Rename Constraint In N Hair Setup





The tool creates dynamic constraints in the N HAIR SETUP. It allows users to select a constraint type and automatically generates a dynamicConstraint node with a clear, consistent, and unique naming convention. The node is automatically renamed and grouped under constraint_GRP.


Usage

Select a dynamic constraint type via the UI checkboxes, then select the object(s) in the scene. Click "Apply" to create the dynamicConstraint node.

Constraint

Exclude Collision Constraint (EC) – Disables collision.

Component To Component Constraint (CTC) – Creates a Stretch or Bend Component constraint.

Force Field Constraint (FF) – Use nConstraints to limit or attach Nucleus objects.

Point to Surface Constraint (PTS) – Constraints points to a surface.

Slide on Surface Constraint (SOS) – Allows constrained points to slide along a surface.

Transform Constraint (TC) – Attaches points to an object transform.

Save And Load HairSystem Attributes





The tool provides a way to export and reapply Maya hairSystem keyable attributes using JSON files. It enables reliable preset management, cross-scene consistency, and safe parameter backup.


Usage

Select a HairSystem in the UI to save its attributes as a preset or to load a preset onto it.


Save HairSystem to JSON

Select a valid HairSystem node in the Maya scene, set the tool option to Save, and execute the tool. Choose a destination and file name for the JSON file, and the tool will export all keyable attributes from the selected hairSystem into the JSON file. A confirmation message will appear in the Maya viewport once the save operation is complete.


Load JSON into HairSystem

Select the target hairSystem node in the Maya scene, set the tool option to Load, and execute the tool. Choose a previously saved JSON file, and the tool will apply all compatible attributes from the JSON file to the selected hairSystem. Any attributes that do not exist or cannot be set are safely skipped with a warning.




Create Yeti Guides Constructions





The tool creates and manages Yeti guide constructions, streamlining the process of connecting Groom nodes to guide curves by automatically generating the necessary Yeti nodes, setting attributes, and building graph connections in pgYetiGraph.


Usage

Select a ‘Groom’ and ‘Guide Curve Set’.


Selected Groom

Click the button and select a single pgYetiGroom node in the scene. The selected Groom node will automatically populate the corresponding text field.


Select Guide Curve Set

Click the button and select a yetiGuideSet_* in the scene. The yetiGuideSet_* chosen is validated to ensure it contains valid NURBS curve members. The selected yetiGuideSet_* will automatically populate the corresponding text field.

The yetiGuideSet_* is configured with standardised attributes:

maxNumberOfGuideInfluences = 1 / stepSize = 0.1 / minimumNumberOfSteps = 10

For each guide curve in the yetiGuideSet_*:

‘saveGuidesRestPosition’=True

weight = 1 / innerRadius = 0 / outerRadius = 0.1 / baseAttraction = 1 /tipAttraction = 1
